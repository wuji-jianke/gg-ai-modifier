## AI Function Calling 系统

AI 对话支持 8 个内存操作工具，LLM 可直接调用真实内存引擎：

| 工具名 | 功能 | 对应 MemoryEngine 方法 |
|--------|------|----------------------|
| `search_memory` | 精确/范围/AOB 搜索 | `searchExact` / `searchByRange` / `searchAob` |
| `read_memory` | 读取指定地址值 | `readMemory` |
| `write_memory` | 写入值到地址（自动回读验证） | `writeMemory` |
| `fuzzy_search` | 模糊搜索（增大/减小/变化/不变） | `searchFuzzy` |
| `filter_results` | 在上次结果中二次过滤 | `filterResults` |
| `save_address` | 收藏地址到书签 | `saveFavorite` |
| `freeze_value` | 冻结/解冻地址值 | `toggleBookmarkFreeze` |

**工作流程**：用户自然语言 → LLM 判断调用工具 → 执行 MemoryEngine 真实操作 → 返回结果 → LLM 生成最终回复

**支持的 LLM**：DeepSeek-V3/R1、OpenAI GPT-4o/4/3.5、小米 MiMo 等兼容 OpenAI API 的模型

## 对话渲染系统

AI 回复通过 WebView 渲染，支持三种内容混合输出：

### 渲染管线

```
原始 Markdown → protectLaTeX() 保护 LaTeX → marked.parse() 渲染 Markdown
    → restoreLaTeX() 恢复占位符 → Prism.highlight() 代码高亮
    → katex.render() 渲染公式 → mermaid.run() 渲染图表
```

### 核心特点

- **需要 Root 权限**：使用独立的 C++ 可执行文件通过 `su` 以 Root 身份读写目标进程内存
- **双模式架构**：主应用 (Flutter) + 悬浮窗 (原生 Android View)，通过 SharedPreferences 同步状态
- **AI Function Calling**：AI 可直接调用 8 个内存操作函数，实现"说一句话就改数据"
- **Markdown/Mermaid/LaTeX 渲染**：AI 回复支持代码块高亮、Mermaid 图表、LaTeX 公式
- **对话渲染系统**：katex@0.16.9/auto-render.min.js,katex@0.16.9/katex.min.js,katex@0.16.9katex.min.css,mermaid@8.14.0/mermaid.min.js,marked@4.3.0/marked.min.js
- **悬浮窗系统**：HTML 渲染管线（marked + KaTeX + Prism.js + mermaid），通过 WebView 渲染

### 本地 JS/CSS 资产

| 库 | 版本 | 文件 |
|----|------|------|
| **KaTeX** | `0.16.9` | `katex.min.js` + `auto-render.min.js` + `katex.min.css` |
| **Mermaid** | `8.14.0` | `mermaid.min.js` |
| **Marked** | `4.3.0` | `marked.min.js` |
| **Prism.js** | `1.29.0` | `prism.min.js` + 16 种语言组件 + `prism-tomorrow.min.css` |

支持的语言高亮：JavaScript、Python、Java、Kotlin、C/C++、C#、Lua、Bash、SQL、JSON、YAML、XML 等

### 两种渲染模式

| 模式 | 实现 | 场景 |
|------|------|------|
| **单内容模式** | `MarkdownWebView(markdownContent: ...)` | 悬浮窗 AI 对话 |
| **多消息模式** | `MarkdownWebView(messages: ...)` | 对话记录详情页（单 WebView 渲染全部消息，避免多 WebView 性能问题） |

## 内存搜索引擎

### Root 架构

```
MemoryEngine.kt
    ↓ JSON 命令 (stdin)
RootScanner.kt → su → scanner_root (C++ 可执行文件)
    ↓ JSON 结果 (stdout)
MemoryEngine.kt → 返回搜索结果
```

### 搜索模式

| 模式 | 方法 | 说明 |
|------|------|------|
| 精确搜索 | `searchExact(value, type)` | 搜索精确数值 |
| 范围搜索 | `searchByRange(min, max, type)` | 搜索数值范围 |
| AOB 搜索 | `searchAob(pattern)` | 按机器码特征码搜索 |
| 模糊搜索 | `searchFuzzy(comparison, type)` | 增大/减小/变化/不变 |
| 结果过滤 | `filterResults(addresses, value, type)` | 在已有结果中二次过滤 |

### 数据类型

| 类型 | 常量 | 字节数 |
|------|------|--------|
| BYTE | `gg.TYPE_BYTE` | 1 |
| WORD | `gg.TYPE_WORD` | 2 |
| DWORD | `gg.TYPE_DWORD` | 4 |
| QWORD | `gg.TYPE_QWORD` | 8 |
| FLOAT | `gg.TYPE_FLOAT` | 4 |
| DOUBLE | `gg.TYPE_DOUBLE` | 8 |
